# Designer Reference

## 목적
- 이 문서는 ASCII 기반 UI 정리에 필요한 최소 기준만 담는다.
- 외부 디자인 툴 기준이 아니라 문서 가독성과 상태 전달을 우선한다.

## ASCII 표기 권장
- 버튼: `[확인]`
- 보조 버튼: `[취소]`
- 입력: `[이메일 입력____________]`
- 검색: `[검색어] [검색]`
- 체크박스: `[ ]`, `[x]`
- 라디오: `( )`, `(o)`
- 탭: `[홈] [탐색] [설정]`
- 카드/패널:
  - `+----------------------+`
  - `| 제목                 |`
  - `| 설명                 |`
  - `+----------------------+`

## 화면 구성 원칙
- 위에서 아래로 읽히게 쓴다.
- 먼저 보이는 것, 나중에 보이는 것을 섞지 않는다.
- 상태는 별도 블록으로 나눈다.
- 모바일과 웹은 필요하면 ASCII를 분리한다.
- 모달, 드로어, 토스트는 본문 화면과 구분한다.

## 리뷰 포인트
- CTA가 분명한가
- 입력과 결과의 관계가 보이는가
- 오류/빈 상태/로딩이 빠지지 않았는가
- 너무 많은 요소가 한 블록에 몰리지 않았는가

## 문서 수정 포인트
- `ASCII 화면`
- `요소 디스크립션`
- `동작·상태`
- 필요하면 `메모`
