# 기획서(A-planning-doc.md) 표준 섹션

validator 가 파싱할 수 있도록 **정확히 아래 한국어 헤딩명을 사용**한다.
다른 표기("Not in scope", "범위 외", "포함하지 않는 것" 등) 금지.

## 필수 섹션 (8개)

| # | 표준 헤딩 | 설명 |
|---|---|---|
| 1 | `## 프로젝트 개요` | 목적, 타겟 사용자, 핵심 가치 |
| 2 | `## 우선순위` | P0 / P1 / P2 구분 |
| 3 | `## 기능 명세` | 각 기능의 동작 정의 |
| 4 | `## 비범위` | 이번 릴리스에서 하지 않는 것 (빈 섹션 금지) |
| 5 | `## 제약/의존성` | 기술/비기술 제약, 외부 의존성 |
| 6 | `## 화면 목록` | screen_id 표 |
| 7 | `## 흐름도` | Mermaid 다이어그램 |
| 8 | `## API / DB 개요` | 필요시. 해당 없으면 "해당 없음" 명시 |

## 파싱 규칙
- `##` 헤딩 레벨 고정
- 헤딩 바로 밑 본문이 비어 있으면 섹션 미작성으로 판정
- "비범위" 섹션은 반드시 최소 1개 이상 항목 포함 (P0 가 아닌 것이 무엇인지 명시)
