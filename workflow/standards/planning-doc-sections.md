# 사이클 계획 md 표준 섹션

이 파일은 `workspace/planning/plan_{item_id}_{short_title}.md` 구조의 기준이다.

## 권장 섹션

| # | 섹션 | 설명 |
|---|---|---|
| 1 | `## 사용자 원문` | 사용자가 실제로 요청한 문장 |
| 2 | `## 목표` | 이번 item에서 달성할 것 |
| 3 | `## 기능 명세` | 동작, 입력, 출력, 규칙 |
| 4 | `## 화면/플로우 요약` | 화면 목록 또는 주요 흐름 |
| 5 | `## ASCII 화면` | 구조와 상태를 보여주는 ASCII |
| 6 | `## 요소 디스크립션` | 각 요소를 사용자 기반 UIUX 관점에서 기술 (종류·선택방식·기본값·필수여부, 색·강조·위계 등 시각 의도 등등) |
| 7 | `## 동작·상태` | 성공, 실패, 로딩, 빈 상태 등 |
| 8 | `## 실행·검증 메모` | 실행 가능한 프로젝트일 때 item 범위 URL/명령/spec 메모 |
| 9 | `## 완료 기준` | 검증 가능한 완료 조건 |
| 10 | `## 비범위` | 이번 사이클에서 하지 않는 것 |
| 11 | `## 메모` | 가정, 후속 참고, 링크 |

## 작성 원칙
- 제목은 파일 맨 위 `#` 하나로 쓴다.
- 본문 섹션은 `##`를 사용한다.
- `ASCII 화면`에는 최소 1개 이상의 fenced code block을 넣는다.
- `요소 디스크립션`은 사용자 기반 UIUX 관점으로 쓴다. 입력/선택 요소는 종류·선택방식·기본값·필수여부를, 시각 요소는 색·강조·위계 같은 의도를 적는다. 나열 항목은 최소 예시일 뿐, 사용자가 쓰기 좋게 하는 데 필요한 UIUX를 폭넓게 적는다.
- 전체 화면에서 색·간격·버튼 등 스타일을 통일한다. 화면마다 제각각으로 두지 않는다.
- 실행 가능한 프로젝트라면 `실행·검증 메모`에 프론트/서버 명령, URL, smoke/full/diag 명령, item 범위 spec 또는 grep 을 적는다.
- Playwright를 쓰는 repo라면 `baseURL`, `webServer`, `projects` 분리 여부, 스크린샷 경로도 적는다.
- 아직 확인되지 않은 실행 값은 추측하지 말고 `미정`이라고 적는다.
- `완료 기준`은 체크 가능한 문장으로 쓴다.
- `비범위`가 없으면 빈 섹션 대신 `없음`이라고 적는다.
