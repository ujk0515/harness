10. 작업명 규칙 + 역할별 모드

모든 Agent 호출 description은 반드시 아래 형식:
[Batch{N}][R{M}][role] mode: 요약
예: [Batch0][R0][planner] plan: 할 일 목록 앱 기획
역할모드설명done ticketmaxTurns
planner plan: 루프 A-1 최초 기획 발급 30
revise: 루프 A-2 / B 재기획 발급 (기존 reopen)
designer review: UX 리뷰만 (design_* 생성 X) 발급 안 함 40
apply: design_* 생성 / 수정 발급
developer review: 루프 B 기술 검토 발급 (review bundle 생성) -
implement: 루프 C 실제 구현 발급
qa review: 루프 B 기획 리뷰 발급 (review bundle 생성) 35
tc: 루프 C 테스트케이스 작성 발급
verify: 루프 D 정적 검증 발급
tester browser verify / re-verify 루프 D Playwright 실행 발급 40
integration test 배포 전 E2E 발급
secretary (모드 없음) 루프 종료 시 기록 발급 -
게이트 동작: validator는 description에서 batch_id, item_id, role, mode를 정규식 파싱한다. mode prefix가 빠지면 PreToolUse에서 차단된다. review: 모드는 작업 자체가 다르므로 별도 분기(예: designer review는 done ticket 발급 안 함, planner revise는 기존 done ticket reopen).
